1. Classes

Classes

Let's say you are organizing your video game library and need to store information for each of the games you own. You can create a new object with individual properties, but if they all share the same ones, it makes sense to create a more efficient way to create and store them. This is where CLASSES come in.

Classes are a blueprint for creating objects with predefined properties and methods. Introduced in ECMAScript 6 (ES6), classes provide a more structured and cleaner syntax to define constructor functions and prototype inheritance, which were previously achieved using functions and prototypes. Picture them as a template to easily plop in the information you want to store for each objects properties, and methods as an efficient way to access, manipulate or create attributes.

Key Concepts

Class Declaration

A class is declared using the class keyword followed by the class name. The body of the class is defined within curly braces {}.

class MyClass {
    // Class body
}
Example
class VideoGame {
}

Creating An Instance

To create an instance of a class, use the new keyword followed by the class name and any required parameters for the constructor.

const myInstance = new MyClass();
Example
const newGame = new VideoGame();

Constructor Method

The constructor method is a special method for creating and initializing an object created with a class. It is called automatically when a new instance of the class is created.

class MyClass {
    constructor(param1, param2) {
        this.property1 = param1;
        this.property2 = param2;
    }
}
Example
class VideoGame {
    constructor(name, developer, year) {
        this.name = name;
        this.developer = developer;
        this.year = year
    }
}

const redDeadRedemption2 = new VideoGame('Red Dead Redemption 2', 'Rockstar Games', '2018')

When it comes to Encapsulation, Getters/Setters can be used to control access to an object's properties, providing a way to encapsulate the internal representation of the data and manage how it is accessed or modified. They provide encapsulation, data validation, computed properties, read-only/write-only properties, and abstraction, thereby enhancing the robustness and maintainability of your code. 

class VideoGame {
    constructor(name, developer, year) {
        this._name = name;
        this._developer = developer;
        this._year = year
    }

    get name() { return this._name; }
    get developer() { return this._developer; } 
    get year() { return this._year; }
    
    set name(newName) { this._name = newName;}
    set developer(newDeveloper) { this._developer = newDeveloper; }
    set year(newYear) { this._year = newYear; }

}

const redDeadRedemption2 = new VideoGame('Red Dead Redemption 2', 'Rockstar Games', '2018');

redDeadRedemption2._name; // Returns this._name
redDeadRedemption2.name; // Calls the get name() property which returns _name

redDeadRedemption2._year = '2020'; // Updates this._year
redDeadRedemption2.year = '2020'; // Calls the set year() property which updates _year

In this instance, the getters/setters aren't assisting with privacy and work the same as a direct variable call. One HUGE benefit of getters/setters is when restricting and enforcing PRIVATE FIELDS. Introduced in ECMAScript 2022, you can explicitly mark an attribute as private using the newer '#' syntax.

The underscore prefix ('_' in JavaScript) utilized in the above code is a convention used to indicate that the property is intended to be private, even they aren't aren't inherently private. It is used to show that the properties should be accessed through getters/setters 

class VideoGame {

    #name;
    #developer;
    #year;

    constructor(name, developer, year) {
        this.#name = name;
        this.#developer = developer;
        this.#year = year
    }

    get name() { return this.#name; }
    get developer() { return this.#developer; } 
    get year() { return this.#year; }
    
    set name(newName) { this.#name = newName;}
    set developer(newDeveloper) { this.#developer = newDeveloper; }
    set year(newYear) { this.#year = newYear; }

}

const redDeadRedemption2 = new VideoGame('Red Dead Redemption 2', 'Rockstar Games', '2018');

redDeadRedemption2.#name; // Will provide a SyntaxError as it cannot be directly accessed
redDeadRedemption2.name; // Calls the get name() method which returns #name

redDeadRedemption2.#year = '2020'; // Will cause an error as it cannot be directly updated
redDeadRedemption2.year = '2020'; // Calls the set year() property which updates #year

Getters can also be used to create Computed Properties that derivce their value from other properties. It gives you the ability to define properties that can dynamically calculate their values based on other data within the object.

class Ammo {
    constructor(bullets, mags) {
        this.bullets = bullets;
        this.mags = mags;
    }

    get count() {
        return this.bullets * this.mags;
    }
}

const rayGunAmmo = new Ammo(12,6);
console.log(rayGunAmmo.count); // This would return the count property, which in this case is 72 (12*6)

Methods

Methods are functions defined inside the class body. They define the behavior of the objects created from the class.

class MyClass {
    constructor(param1, param2) {
        this.property1 = param1;
        this.property2 = param2;
    }

    method1() {
        console.log(this.property1);
    }

    method2() {
        console.log(this.property2);
    }
}
Example
class VideoGame {
    constructor(name, developer, year) {
        this.name = name;
        this.developer = developer;
        this.year = year
    }

    strOutput() {
        return `${this.name} was developed by ${this.developer} and released in ${this.year}.`;
    }
}

const redDeadRedemption2 = new VideoGame('Red Dead Redemption 2', 'Rockstar Games', '2018');

console.log(redDeadRedemption2.strOutput()); // Output: Red Dead Redemption 2 was developed by Rockstar Games and released in 2018.

Static Methods

Static methods are methods defined on the class itself, rather than on instances of the class. They are called on the class and not on the objects created from the class. Static methods are often used to create utility functions that are related to the class but do not operate on instances of the class.

class HitDMG {
    static headDMG(hits, multiplier) {
        return hits * multiplier;
    }

    static limbDMG(hits, multiplier) {
        return hits * multiplier;
    }
}

HitDMG.headDMG(3, 1.4) // Output: 4.2 (3 hits * 1.4 multiplier)
HitDMG.limbDMG(3, 0.8) // Output: 2.4 (3 hits * 0.8 multiplier)

With the static method, a new instance of HitDMG didn't need to be created to utilize, as it was called directly on the class and not an INSTANCE of the class. Due to this, a static method would not be able to access data of an object, and can only access other static methods or properties. These are useful for utility and operations that aren't tied to a specific object or instance.
