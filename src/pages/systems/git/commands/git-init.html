1. init

git init

The git init command is used to create a new Git repository. It can be used to initialize a new, empty repository or reinitialize an existing one. Before executing the git init command, Git does not track any files on the system. Tracking begins only after the repository is initialized with git init. From that point, you can commit files and push them to a remote repository.

Usage

git init
git init [options] [directory]

Without any option or directory specifications, the command will create a repository within the current working directory.

Options

--q, --quiet

git init --q --quiet

git init --q example_repo
git init --quiet

Used to suppresses the output that is typically displayed when a new repository is initialized. This can be useful if you want to initialize a repository without any verbose messages cluttering your console or script output.

--bare

git init --bare

git init --bare example_repo.git

Creates a bare repository. A bare repository contains only the version control information without a working directory. It is typically used as a central repository for collaboration.

--object-format

git init --object-format=<format>

git init --object-format<format>=sha1 example_repo
git init --object-format<format>=sha256 example_repo

Specifies the format of the Git objects that the repository will use. Git objects, such as commits, trees, blobs, and tags, are the fundamental components of a Git repository. This option allows you to choose between different object formats. The primary object formats currently supported are SHA-1 (traditional format using the SHA-1 hashing algorithm) and SHA-256 (A newer format using the SHA-256 hashing algorithm, providing better security).

--initial-branch

git init --initial-branch=<branch-name>

git init --initial-branch=main
git init --initial-branch=main example_repo

Allows you to specify the name of the initial branch that is created when you initialize a new Git repository.

--template=<template-directory>

git init --template=<template-directory>

git init --template=/path/template

Specifies a directory containing template files for the repository. These template files can include hooks, configuration files, and other initialization scripts.

--separate-git-dir=<git-dir>

git init --separate-git-dir=<git-dir>

git init --separate-git-dir=/path/git-dir

Creates the repository's Git directory as a separate directory from the working directory. This is useful for placing the repository's data in a different location.

--shared[=<permissions>]

git init --shared[=umask|group|all]

git init --shared[=umask]
git init --shared[=group]
git init --shared[=all]

Specifies that the repository will be shared among several users. This option sets the directory permissions to be group-writable and makes the repository suitable for collaborative use. The <permissions> argument can be umask, group, or all
